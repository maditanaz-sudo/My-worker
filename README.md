# MHR-CFW · Cloudflare Worker Deployment via GitHub

<div dir="rtl">

## 🧩 مسئله

دسترسی به **Cloudflare Dashboard** و **ویرایشگر Worker** در برخی مناطق (مانند ایران) با محدودیت یا اختلال مواجه است.  
این پروژه این مشکل را حل می‌کند: شما می‌توانید Worker خود را مستقیماً از **GitHub** دپلوی کنید، بدون اینکه نیاز به باز کردن ویرایشگر Cloudflare داشته باشید.

---

## 🚀 قابلیت‌ها

- دپلوی خودکار Worker با هر بار `push` به مخزن GitHub
- بدون نیاز به ویرایشگر Cloudflare
- مناسب برای **MHR-CFW** (HTTP Relay)
- امنیت با `AUTH_KEY` (قابل تنظیم توسط کاربر)
- کاملاً **اوپن سورس** و قابل **Fork** توسط دیگران

---

## 📦 پیش‌نیازها

- یک حساب **Cloudflare** (رایگان کافیست)
- یک حساب **GitHub**
- توانایی گرفتن **API Token** از Cloudflare (با استفاده از **سرویس شکن** یا VPN در صورت نیاز)

---

## 🛠️ مراحل استفاده (برای کاربران جدید)

### ۱. **Fork** کردن مخزن

این مخزن را **Fork** کنید تا یک کپی در حساب خود داشته باشید.

### ۲. تغییر `AUTH_KEY` در `src/worker.js`

برای امنیت، در مخزن خودتان مقدار پیش‌فرض `AUTH_KEY` را به یک رمز دلخواه و قوی تغییر دهید:

```javascript
const AUTH_KEY = "my_strong_secret_key_here";  // این را عوض کنید
```

> **⚠️ هشدار امنیتی:** این کلید را در مخزن عمومی قرار ندهید. در بخش بعدی راه درست را توضیح می‌دهیم.

---

### ۳. دریافت توکن از Cloudflare

برای اینکه Worker روی Cloudflare شما دپلوی شود، به دو چیز نیاز دارید:

1. **API Token** با دسترسی `Edit Cloudflare Workers`
2. **Account ID**

اگر خودتان به Dashboard دسترسی ندارید، از **سرویس شکن** یا VPN استفاده کنید تا به Cloudflare Dashboard وصل شوید و این موارد را بگیرید.

---

### ۴. تنظیم GitHub Secrets

در مخزن Fork شده خود:

- به **Settings** → **Secrets and variables** → **Actions** بروید
- دو Secret اضافه کنید:

| نام Secret | مقدار |
|------------|--------|
| `CLOUDFLARE_API_TOKEN` | توکنی که از Cloudflare گرفتید |
| `CLOUDFLARE_ACCOUNT_ID` | Account ID خود (از Cloudflare Dashboard) |

---

### ۵. (اختیاری) تنظیم Environment Variable برای `AUTH_KEY`

برای امنیت بیشتر، به جای گذاشتن `AUTH_KEY` در کد، آن را در **Cloudflare Worker** به عنوان **Environment Variable** (متغیر محیطی) اضافه کنید:

- در Cloudflare Dashboard به **Workers & Pages** → **Worker شما** → **Settings** → **Variables and Secrets** بروید
- یک متغیر به نام `AUTH_KEY` با مقدار واقعی اضافه کنید
- در `src/worker.js` کد را به صورت زیر تغییر دهید:

```javascript
const AUTH_KEY = process.env.AUTH_KEY || "default_placeholder";
```

---

### ۶. دپلوی خودکار

هر بار که فایل `src/worker.js` را تغییر دهید و **Commit** کنید، GitHub Actions خودکار Worker شما را روی Cloudflare دپلوی می‌کند.

---

## 🔐 نکات امنیتی مهم

- **هرگز** `AUTH_KEY` واقعی را در یک مخزن عمومی قرار ندهید.
- از **Environment Variables** در Cloudflare برای ذخیره `AUTH_KEY` استفاده کنید.
- **آدرس Worker** خود را مخفی نگه دارید. اگر آدرس و `AUTH_KEY` هر دو فاش شوند، هر کسی می‌تواند از Worker شما استفاده کند.
- از یک رمز قوی برای `AUTH_KEY` استفاده کنید (حداقل 20 کاراکتر، ترکیبی از حروف و اعداد).
